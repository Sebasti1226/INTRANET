📋 Tipos de Reporte - Guía de Clasificación

Usa esta guía para identificar rápidamente el tipo de caso y saber a qué equipo escalarlo.

🔴

Fibra Oscura

🚨 Corte de Fibra

Descripción: Enlace caído, sin luz, pérdida total de servicio

Equipo: Torre de Control

Prioridad: ALTA

Nota: Solicitar tramos afectados y hora exacta del corte

⚠️ Degradación

Descripción: Servicio intermitente, pérdida de paquetes, latencia alta

Equipo: Torre de Control

Prioridad: MEDIA

Nota: Solicitar gráficas de monitoreo y horario de afectación

🌐

Red Complementaria

📡 Conectividad / Internet

Descripción: Sin acceso a Internet, problema de navegación

Equipo: NOC - Red Complementaria

Prioridad: ALTA

Nota: Validar IP pública, gateway y DNS

🔧 Configuración de Equipos

Descripción: Cambios en router, firewall, VLANs

Equipo: NOC - Red Complementaria

Prioridad: MEDIA

Nota: Requiere autorización del cliente y configuración previa

⚡

Alarmas IFX

🔔 Alarma Activa

Descripción: Cliente reporta alarma en NMS/sistema de monitoreo

Equipo: Torre de Control

Prioridad: ALTA

Nota: Solicitar captura de pantalla de la alarma y código

📊 Revisión de Histórico

Descripción: Consulta sobre alarmas pasadas

Equipo: Torre de Control

Prioridad: BAJA

Nota: Indicar rango de fechas a revisar

🔑

Accesos

🚪 Solicitud de Acceso a SSEE

Descripción: Autorización para ingreso a subestaciones

Equipo: ETCEE (Guatemala)

Prioridad: MEDIA

Nota: Usar módulo de Accesos. Incluir: SSEE, horario y personal

🏢 Acceso a Datacenters

Descripción: Ingreso a DC o facilities del cliente

Equipo: PM/KAM del cliente

Prioridad: MEDIA

Nota: Coordinar con 24-48h de anticipación

🔐

Credenciales

🆕 Creación de Credenciales

Descripción: Cliente nuevo solicita acceso al portal

Equipo: Service Desk (Propio)

Prioridad: BAJA

Nota: Tiempo de gestión: 72 horas. Requiere datos del cliente

🔄 Recuperación de Credenciales

Descripción: Cliente olvidó usuario/contraseña

Equipo: Service Desk (Propio)

Prioridad: BAJA

Nota: Validar identidad del solicitante

📄

Casos Administrativos

💰 Facturación / Cartera

Descripción: Consultas sobre pagos, facturas, suspensiones

Equipo: Área de Cartera / Finanzas

Prioridad: BAJA

Nota: Service Desk NO gestiona. Derivar a área comercial

📋 Solicitud de RFO

Descripción: Cliente solicita reporte formal de falla

Equipo: Área de Calidad

Prioridad: MEDIA

Nota: Tiempo de entrega: 48-72 horas hábiles

🤝 Solicitudes PM/KAM

Descripción: Cambios contractuales, ampliaciones, cancelaciones

Equipo: PM o KAM asignado

Prioridad: BAJA

Nota: Service Desk NO gestiona. Cerrar ticket y derivar

🔧

Mantenimientos

📅 Mantenimiento Programado

Descripción: Trabajos preventivos con ventana autorizada

Equipo: Torre de Control / O&M

Prioridad: MEDIA

Nota: Notificar al cliente con 48-72h de anticipación

⚡ Mantenimiento Emergente

Descripción: Trabajos urgentes sin ventana programada

Equipo: Torre de Control

Prioridad: ALTA

Nota: Informar al cliente inmediatamente del impacto

💡

Consultas Técnicas

📊 Gráficas de Consumo

Descripción: Solicitud de estadísticas de tráfico/BW

Equipo: Torre de Control

Prioridad: BAJA

Nota: Especificar rango de fechas y circuito

🗺️ Información de Ruta

Descripción: Consulta sobre trazado físico del enlace

Equipo: Ingeniería / PM

Prioridad: BAJA

Nota: Información sensible, validar autorización

✅ Validación de Operatividad

Descripción: Cliente solicita confirmar estado del servicio

Equipo: Torre de Control / NOC

Prioridad: BAJA

Nota: Revisar monitoreo y confirmar al cliente

🏗️

Construcción

🆕 Servicio en Construcción

Descripción: ID aún no activo, en fase de implementación

Equipo: Project Manager

Prioridad: MEDIA

Nota: Service Desk NO gestiona. Derivar a PM del proyecto

📡 Instalación de Equipos

Descripción: Coordinación para instalación CPE

Equipo: Instalaciones / PM

Prioridad: MEDIA

Nota: Coordinar horario con cliente

🎯 Leyenda de Prioridades

ALTA

Servicio caído o afectación crítica - Respuesta inmediata

MEDIA

Afectación parcial o coordinación requerida - Respuesta en 2-4h

BAJA

Consultas, solicitudes administrativas - Respuesta en 24-48h

📊 CUADROS DE CIERRE

Estos son los cuadros estándar para cierre de tickets. Todos son NO IMPUTABLES

🔄 REESTABLECE SIN INTERVENCIÓN

Causa: Cliente / No determinada / Desconocida - Reestablece sin intervención

Solución: Sin acciones correctivas

Imputabilidad: NO IMPUTABLE

💤 SIN RESPUESTA

Causa: Cliente / No Aplica / Sin accesos o respuesta

Solución: Sin acciones correctivas

Imputabilidad: NO IMPUTABLE

❌ MAL REPORTE

Causa: Cliente / No Aplica / Mal reporte

Solución: No aplica

Imputabilidad: NO IMPUTABLE

🔧 VM (VENTANA DE MANTENIMIENTO)

Causa: Cliente / No Aplica / VM Notificada

Solución: VM Notificada

Imputabilidad: NO IMPUTABLE

🔁 TICKET DUPLICADO

Causa: Cliente / No Aplica / Ticket Duplicado

Solución: Se maneja con el TT [número del ticket master]

Imputabilidad: NO IMPUTABLE

📊 GRÁFICAS DE CONSUMO

Causa: Cliente / Solicitudes / Gráficas

Solución: Se procede con el envío de las gráficas

Imputabilidad: NO IMPUTABLE

🔑 ACCESOS DATACENTER

Causa: Cliente / Solicitudes / Otras / Accesos Datacenter

Solución: Accesos autorizados

Imputabilidad: NO IMPUTABLE

🌐 RED COMPLEMENTARIA

Causa: Cliente / No aplica / Mal Reporte

Solución: Ninguna por parte de Ufinet - Falta de plantilla complementaria y registro fotográfico

Imputabilidad: NO IMPUTABLE

🤝 SOLICITUD PM/KAM

Causa: Cliente / Solicitudes / Solicitud área comercial/PM

Solución: Se traslada al cliente, al área Comercial/PM

Imputabilidad: NO IMPUTABLE

⏸️ SERVICIO SUSPENDIDO

Causa: Cliente / No aplica / Servicio Suspendido

Solución: No aplica - Se redirige al cliente al área de cartera

Imputabilidad: NO IMPUTABLE

⚠️ VALIDACIONES OBLIGATORIAS

  • ✅ Todos los ID terminados en E, X deben ser validados primero por N1
  • ✅ NO escalar directamente a TORRE sin validar con N1 primero
  • ✅ Cuando hay casos resueltos por una master, SI O SI toca crear nuevo caso imputable a Ufinet
